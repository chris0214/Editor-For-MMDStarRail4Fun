# Design System

## Overview

这是一个面向 MMD 材质作者的桌面工作台。视觉参照是调色、节点合成和代码检查工具：画布承担固定阶段关系，检查器承担精确输入，源码视图承担改动反馈，状态栏承担结果反馈。

**画布优先规则。** 节点关系始终是主视图，参数面板不能挤压画布到无法阅读。

## Colors

底色使用中性深灰 `24,25,28`，面板逐级提升到 `31,32,36` 和 `39,40,45`。主操作色是低饱和珊瑚色 `218,112,92`。成功、提醒和错误分别使用绿色、琥珀色和红色。

**单一强调规则。** 珊瑚色只表达选择、焦点和主要操作，节点类别使用低饱和辅助色，禁止发光效果。

## Typography

界面文字使用 Microsoft YaHei UI，源码与参数标识使用 Cascadia Mono。常规界面保持 9pt，面板标题使用 12pt 粗体。字距始终为 0。

**参数可读规则。** 用户名称在左，源码标识在右；任何一侧都不能挤掉真正的输入控件。

## Elevation

层级主要依靠表面亮度和 1px 边界表达。节点圆角为 6px，不使用大面积阴影。浮层沿用系统菜单行为。

**工具表面规则。** 面板不是装饰卡片，禁止卡片嵌套和大圆角容器。

## Components

节点固定为 186x70，最小缩放为 75%，标题、摘要、状态点和输入输出端口尺寸稳定。参数检查器使用单个 WPF 可视树承载，参数行高度由内容稳定决定，不为每个输入创建 WinForms HWND。所有常用按钮至少 30px 高，并提供工具提示。

**渐进展开规则。** 常用模式展示来源、纹理、材质 0 和全局强度；完整模式才展开材质 1-7、UV 变换和底层向量。

## Do's And Don'ts

- 保留高密度，但使用分组、间距和对齐降低扫描成本。
- 每次参数修改立即更新源码改动标记和状态。
- 源码预览只读，改动行同时使用背景和 gutter 字符标记。
- 错误说明必须包含恢复方向，并保留用户输入。
- 禁止紫蓝渐变、霓虹外发光、玻璃拟态和装饰性大标题。
- 禁止用颜色作为唯一状态信号。
- 禁止把固定 Shader 阶段伪装成可以任意连线的自由图。

审计测试：如果界面像营销网站或通用后台，而不是材质制作工具，设计方向就是错误的。
