# StarRail 材质节点编辑器使用教程

StarRailShaderEditor 是面向 `MMDStarRail4Fun_v2.1` 的非官方固定管线材质编辑器。它把 FX 中可由用户调整的参数整理成中文分组、节点和可视控件，并提供阴影参数、只读源码定位、资源检查、控制器 Morph 预设和帧 0 VMD 导出。

程序不附带、下载或解包 Shader。每次启动都由你选择自己合法取得的 `MMDStarRail4Fun` 文件夹，只有在你点击“保存”“另存”或“导出包”后才会写入文件。

## 1. 发布包内容

- `StarRailShaderEditor-v1.0.0-win-x64.exe`：Windows x64 自包含单文件程序，不需要另装 .NET。
- `GUIDE.md`：程序内“使用说明”按钮打开的本教程。
- `StarRailShaderEditor-v1.0.0-使用教程.md`：内容相同的版本化教程文件，便于归档和分享。
- `LICENSE`：编辑器自身源码和自有参数元数据的 MIT 许可证。
- `SHA256SUMS.txt`：用于确认 EXE 是否完整、是否在传输中被修改。

发布包不包含 `.fx`、`.hlsl`、`.pmx`、`.vmd`、纹理或配套 Shader 的其他文件。

## 2. 使用前准备

1. 使用 Windows 10 或 Windows 11 的 64 位版本。
2. 准备一份可以正常使用的 `MMDStarRail4Fun_v2.1` 文件夹。
3. 确认该文件夹根目录中存在 `sr_body.fx`。完整功能还需要其他材质 FX、`Shadow.fx`、`Shadow_zbuffer.fx`、`internal/shader.hlsl` 和 `fun_controller.pmx`。
4. 第一次正式修改前，建议额外复制一份原始 Shader 文件夹留作人工备份。

程序不要求安装 Visual Studio 或 .NET Runtime。Windows 首次运行从网络下载的 EXE 时可能显示安全提示；请先用 `SHA256SUMS.txt` 核对文件，再根据自己的安全策略决定是否运行。

## 3. 第一次启动

1. 双击 `StarRailShaderEditor-v1.0.0-win-x64.exe`。
2. 在“选择文件夹”窗口中选择包含 `sr_body.fx` 的 `MMDStarRail4Fun` 文件夹。
3. 也可以选择它的上一级目录；如果其中存在 `MMDStarRail4Fun` 子目录，程序会自动识别。
4. 如果提示“不是有效的 Shader 文件夹”，请检查是否选到了纹理子目录、模型目录或压缩包所在目录，而不是 Shader 根目录。
5. 取消文件夹选择会直接退出，不会创建或修改任何 Shader 文件。

程序只在本机设置中记住上次目录，方便下次打开选择器时定位；它不会把 Shader 复制到 EXE 所在目录。

## 4. 界面认识

### 顶部工具栏

- “打开”：额外打开一份 StarRail FX 文件。
- “保存”：把当前修改写回原文件，并创建自动备份。
- “另存”：把当前 FX 保存为新文件，不覆盖当前文件。
- “导出包”：预检资源后，生成使用相对纹理路径的便携目录。
- “导出 VMD”：把控制器的 18 个 Morph 写成帧 0 VMD；选择控制器后可用。
- “撤销 / 重做”：恢复或重新应用参数调整。
- “适合画布”：把固定管线节点重新放进可见区域，缩放不会低于 75%。
- “检查资源”：重新检查纹理和核心 Shader 路径，并展开诊断。
- “诊断”：展开或收起底部诊断面板。
- “使用说明”：打开 EXE 旁边的本教程。

### 左侧区域

- “当前文件”：切换六份角色材质、两份阴影 FX、只读参考源码和控制器。
- “搜索参数或源码名”：可输入中文名或 FX 中的源码 ID。
- “参数分组”：按基础颜色、卡通阴影、高光、边缘阴影、星空等功能筛选。
- “显示全部参数”：关闭时优先显示零基础常用项；打开后显示该文档的所有可调项。

### 中央区域

- “节点”：显示固定 StarRail 渲染流程。节点用于导航，不能新增、删除或自由连线。
- “源码”：只读查看当前 FX 或 `internal/shader.hlsl`，支持查找、复制和定位行。
- 底部诊断：集中显示缺失纹理、绝对路径、核心文件缺失和元数据问题。

### 右侧检查器

点击节点或左侧分组后，右侧显示对应参数。中文名是主要标题，下面的等宽文本是原始源码 ID；说明文字会提示用途和单位。

## 5. 零基础推荐流程

1. 在“当前文件”中选择要调整的材质，例如身体、脸部或头发。
2. 保持“显示全部参数”关闭，先只处理常用项。
3. 点击中间节点，再从左侧选择一个参数分组。
4. 每次只小幅修改一个值，观察 MMD 中的实际结果后再继续。
5. 点击参数行右上角的 `>`，检查它对应的 FX 声明是否正确。
6. 点击“检查资源”，先解决红色错误；黄色绝对路径提示建议在分享前处理。
7. 使用 `Ctrl+S` 保存。确认 MMD 中结果正常后，再修改下一组。
8. 需要发给另一台电脑时使用“导出包”，不要直接散落复制纹理和 FX。

编辑器不连接正在运行的 MMD，也不会实时注入 MME。参数保存后，需要按你的 MMD/MME 工作流重新载入效果或刷新场景才能看到结果。

## 6. 参数控件怎么用

### 滑杆与数值框

拖动滑杆适合快速试效果，数值框适合精确输入。一次完整拖动只记为一次撤销；输入超出硬范围的值会被拒绝或限制。

### 向量参数

`X / Y / Z / W` 分量分别输入，各分量可能有不同范围。例如 `BaseColorMapST` 的缩放可以大于 `1`，UV 偏移也可以是负数；它不会被颜色参数的 `0–1` 范围错误夹紧。

### 颜色参数

点击色块打开颜色选择器，也可以精确调整颜色分量。白色乘色通常表示“不改变原颜色”，颜色分量范围为 `0–1`。

### 下拉菜单

用于选择有明确枚举含义的模式或纹理通道。点击后选择一项即可，一次选择对应一次撤销记录。

### 开关

用于真正有 FX Define 支持的阶段或功能。关闭功能后，相关纹理不会作为启用资源参与导出预检。

### 纹理路径

输入框显示 FX 中的路径；使用选择按钮可以从本机选择纹理。绿色表示资源可访问，黄色通常表示路径存在但使用了不适合跨电脑分享的绝对路径，红色表示启用中的资源缺失。

## 7. 常用材质概念

### 基础颜色

- 基础颜色来源可选择 PMX 材质、常量或单独纹理。
- 染色会与原颜色相乘，白色通常表示不改变。
- Gamma 改变明暗分布，建议先在 `0.8–1.3` 附近小幅调整。
- 透明度为 `1` 时不透明，降低后会变透明；同时检查模型材质和 MME 的透明设置。

### 子材质 0–7

LightMap 或 FaceMap 的通道会被转换为 `0–7` 的材质编号。带“材质 0”到“材质 7”的参数分别控制不同区域。初次使用先调整材质 0；确认模型实际使用其他编号后，再打开“显示全部参数”。

### 卡通阴影

- 明暗分界决定亮面和暗面的切换位置。
- 冷色 Ramp、暖色 Ramp 会根据灯光方向混合阴影色带。
- 自阴影使用 `Shadow.fx` 生成的深度图遮挡角色自身。

### 高光

- Shininess 越大，高光通常越集中。
- Roughness 越大，高光边缘通常越柔和。
- Intensity 控制高光强度。
- Threshold 通常由 LightMap 的指定通道提供。

### 边缘阴影

这一组沿观察角度压暗或染色角色轮廓。强度过高会让轮廓显脏，建议先从材质 0 的颜色、宽度和羽化开始。

### 星空效果

在支持的节点上右键可以开启或关闭星空阶段。星点纹理、噪声纹理、密度、景深和 Fresnel 可以组合出服装内部空间感。启用后应点击“检查资源”，确认所有星空相关纹理都存在。

## 8. 节点画布操作

- 单击节点：选中阶段并刷新右侧检查器。
- 拖动节点：调整画布位置，不会改变 Shader 的执行顺序。
- 鼠标中键拖动，或按住空格再拖动：平移画布。
- 鼠标滚轮：缩放画布。
- 双击空白区域或点击“适合画布”：重新适配视图。
- 右键支持的节点：启用或停用确实有 Define 支持的阶段。

节点位置、中心视图和面板尺寸保存在 Shader 文件夹的 `.starrail-editor/session.json`。删除这个会话文件可恢复默认布局，不会改变 FX。

## 9. 源码预览与修改标记

1. 点击中央“源码”，切换到只读 Notepad 式源码视图。
2. 在右侧参数标题旁点击 `>`，可直接跳到参数声明行。
3. 修改参数后，源码视图会自动记住最近修改位置。
4. gutter 中 `M` 表示尚未保存的修改行，`>` 表示当前定位行。
5. 保存成功后，已保存行的修改标记会清除。
6. 使用 `Ctrl+F` 聚焦源码查找框，可查找参数名、Define 或注释。

`internal/shader.hlsl` 只作为参考源码显示，可以搜索和复制，但不能在编辑器中修改。编辑器也不会公开或改写其中的实现级常量。

## 10. 保存、备份与恢复

- `Ctrl+S` 或“保存”：写回当前 FX，并在原文件旁创建 `.starrail-editor-backups`。
- 自动备份最多保留最近 10 次，同一秒连续保存也会生成不同文件。
- “另存”：创建当前 FX 的副本，适合试验另一套参数。
- `Ctrl+Z`：撤销最近一次参数操作。
- `Ctrl+Y`：重做刚撤销的操作。
- 滑杆从按下到松开算一次操作；下拉、开关和纹理选择各算一次操作。

保存采用同目录临时文件替换。发生权限、占用或磁盘错误时，当前未保存修改会保留；解决问题后可以再次保存。若需要人工恢复，可从 `.starrail-editor-backups` 中选择对应时间的备份复制回原位置。

## 11. 资源诊断与便携导出

### 检查资源

点击“检查资源”后：

- 红色错误必须先处理，例如启用中的纹理不存在。
- 黄色提示通常可以继续编辑，但表示绝对路径在其他电脑上可能失效。
- 双击诊断项可定位到对应参数，再重新选择正确纹理。

### 导出包

1. 先保存需要保留的修改。
2. 点击“导出包”，选择一个输出目录。
3. 程序先检查所有启用中的纹理；有任何缺失时会列出来源，并且不会生成最终目录。
4. 检查通过后，程序在临时 staging 中复制所需材质、阴影、核心 Shader、控制器和纹理，并把包内纹理路径改为相对路径。
5. 完成后会打开实际输出目录；如果同名目录已存在，程序会选择唯一目录，避免覆盖旧包。

“导出包”是你对自己已取得资源执行的本地操作。分享导出内容前，必须确认配套 Shader、模型和纹理的许可证允许再分发；编辑器的 MIT 许可证不会自动覆盖这些外部文件。

## 12. Shadow 参数

- `Shadow.fx`：可调整阴影贴图分辨率和后处理合成强度。
- `Shadow_zbuffer.fx`：可调整 `DepthBias`，用于缓解阴影痤疮或漂浮问题。
- 其他阴影实现常量保持只读，避免生成未经验证的组合。

提高阴影贴图分辨率会增加显存和渲染开销。`DepthBias` 应小幅调整：太小可能出现条纹，太大可能让阴影与模型表面分离。

## 13. 控制器 Morph 与 VMD

### 编辑和保存预设

1. 在“当前文件”选择“控制器 Morph”。
2. 右侧会读取 `fun_controller.pmx` 中的模型名和 18 个 Morph。
3. 每个 Morph 权重范围为 `0–1`；修改控制器不会写入原 PMX。
4. 保存后，当前预设写入 `.starrail-editor/controller-presets/current.json`。

### 导出帧 0 VMD

1. 调整好全部 Morph 权重。
2. 点击顶部“导出 VMD”。
3. 选择保存位置，默认文件名为 `fun_controller_preset.vmd`。
4. 程序会写入全部 18 个 Morph，包括值为 0 的项目，帧号均为 0。

### 在 MMD 中加载

1. 打开 MMD，并载入配套的 `fun_controller.pmx`。
2. 在模型操作区域确认当前选中的是 `fun_controller.pmx`，不要选角色模型或相机。
3. 使用 MMD 的动作数据加载功能，选择刚导出的 `.vmd`。
4. 回到第 0 帧，确认控制器 Morph 值已经出现。
5. 播放或渲染前检查角色材质是否已经正确应用配套 FX。

如果 MMD 提示动作数据不适用，优先确认当前选中模型、PMX 版本和控制器是否来自同一套 Shader。不要手工改 VMD 文件名来替代模型匹配。

## 15. 常见问题

### 启动后一直要求选择文件夹

这是正常设计。EXE 不内置 Shader，每次启动都需要确认本机资源目录；上次目录只用于让选择器更快定位。

### 选择文件夹后提示缺少 `sr_body.fx`

请选择 `MMDStarRail4Fun` 根目录或它的上一级目录，不要选择 `internal`、纹理目录、压缩包或编辑器目录。

### 修改后 MMD 没变化

确认已经保存正确的 FX，并在 MMD/MME 中重新载入效果或刷新场景。这个编辑器不会连接或注入正在运行的 MMD。

### 右侧没有参数

确认当前选择的是可编辑 FX 或控制器，再点击一个有参数的节点/分组。`internal/shader.hlsl` 是只读参考文件，不会显示可编辑参数。

### 下拉菜单点不开或界面显示异常

先关闭并重新打开程序，确认使用的是发布包中的最新 EXE。如果问题仍能稳定复现，请记录 Windows 缩放比例、显示器刷新率、当前文件和具体操作步骤。

### 拖动窗口或切换节点不流畅

最新检查器使用 WPF 承载以减少 WinForms 重建卡顿。若仍异常，请先关闭显卡覆盖层、录屏或窗口美化工具做对照，并记录是否只有本程序出现问题。

### 保存失败

检查 FX 是否只读、是否被其他程序独占、所在目录是否允许写入，以及磁盘空间是否充足。失败不会清空当前修改。

### 导出包失败

展开诊断，逐项修复启用中的缺失纹理。资源未通过预检时，程序不会把不完整目录当作成功结果。

### “使用说明”按钮打不开

请确保 `StarRailShaderEditor-v1.0.0-使用教程.md` 或 `GUIDE.md` 与 EXE 位于同一目录；也可以直接用记事本或支持 Markdown 的编辑器打开本文件。
